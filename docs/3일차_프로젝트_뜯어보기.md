# [3일차] 프로젝트 뜯어보기 및 구조 분석

## 1. 각 디렉터리의 역할
* **app/core/**: 프로젝트 전반의 설정, 데이터베이스 연결, 환경 변수 관리 등 핵심 인프라 코드를 담당합니다.
* **app/models/**: 데이터베이스의 테이블 구조를 파이썬 클래스로 정의합니다.
* **app/repositories/**: 데이터베이스와 직접 소통하여 데이터를 조회, 저장, 수정, 삭제하는 작업을 수행합니다.
* **app/schemas/**: API가 주고받는 데이터의 형식을 정의하고 검증합니다(Pydantic).
* **app/services/**: 레포지토리와 외부 API를 조합하여 비즈니스 로직을 처리하는 핵심 공간입니다.
* **app/apis/**: 클라이언트의 요청(Endpoint)을 받고 서비스로 연결하는 라우터 역할을 합니다.

## 2. 주요 파일의 역할
* **app/main.py**: 서버의 시작점으로, 애플리케이션 인스턴스를 생성하고 라우터를 등록합니다.
* **app/core/config.py**: 환경 변수(DB 접속 정보, API Key 등)를 관리하고 설정값을 읽어옵니다.
* **pyproject.toml**: 프로젝트에 필요한 라이브러리와 의존성 설정을 관리합니다.
* **uv.lock**: 설치된 라이브러리들의 버전을 고정하여 환경의 일관성을 유지합니다.

## 3. 데이터베이스 연결
* **app/core/db/databases.py**: 데이터베이스 엔진을 생성하고, 서비스에서 사용할 DB 세션을 연결하는 전용 통로 역할을 합니다.

## 4. 모델 작성 및 마이그레이션
* **모델 작성**: `app/models/`에 SQL 테이블 구조를 클래스로 선언합니다.
* **마이그레이션**: `alembic`을 사용하여 모델 변경 사항을 DB 스키마에 자동으로 적용(revision -> upgrade)합니다.

## 5. API 구현 흐름
1. 클라이언트가 API 호출
2. `apis/`에서 요청 수신 및 `schemas/`로 유효성 검사
3. `services/`에서 비즈니스 로직 처리
4. 필요 시 `repositories/`를 통해 DB 연동
5. 최종 응답 반환
